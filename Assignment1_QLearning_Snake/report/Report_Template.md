## 1. 引言 (约150字)

本项目旨在使用Q-Learning强化学习算法训练一个智能体玩贪吃蛇游戏。强化学习是机器学习的重要分支，通过智能体与环境的交互学习最优策略。贪吃蛇游戏作为经典的游戏案例，具有清晰的目标（吃食物、避免碰撞）和简单的规则，非常适合作为强化学习的入门项目。

## 2. 游戏设计 (约300字)

### 2.1 游戏规则

本项目实现的贪吃蛇游戏遵循以下规则：
- 游戏在10×10的网格地图上进行
- 蛇初始长度为3个单位，初始位置在地图中心
- 食物随机出现在空白格子中
- 蛇吃到食物后长度+1，得分+1
- 蛇撞墙或撞到自己身体则游戏结束
- 如果蛇在100×当前长度步数内未吃到食物，游戏也会结束（防止无限循环）

### 2.2 状态空间定义

状态空间是智能体感知环境的方式。本项目设计了**11维状态向量**：

**危险检测（3维）**：
- 前方是否有危险（墙壁或蛇身）[0/1]
- 右侧是否有危险 [0/1]
- 左侧是否有危险 [0/1]

**移动方向（4维）**：
- 当前是否向左移动 [0/1]
- 当前是否向右移动 [0/1]
- 当前是否向上移动 [0/1]
- 当前是否向下移动 [0/1]

**食物位置（4维）**：
- 食物是否在左边 [0/1]
- 食物是否在右边 [0/1]
- 食物是否在上边 [0/1]
- 食物是否在下边 [0/1]

这种状态表示方式的优点是：(1) 维度较低，便于Q表存储；(2) 包含关键信息（危险、方向、目标）；(3) 使用相对位置而非绝对坐标，提高泛化能力。

### 2.3 动作空间定义

动作空间包含**3个动作**（使用one-hot编码）：
- `[1, 0, 0]`: 直行（保持当前方向）
- `[0, 1, 0]`: 右转（相对当前方向顺时针转90度）
- `[0, 0, 1]`: 左转（相对当前方向逆时针转90度）

这种设计相比直接使用上下左右四个方向的优势在于：避免了蛇180度掉头的非法操作，使动作空间更加合理。

### 2.4 奖励函数设计

奖励函数是强化学习的核心，直接影响学习效果。本项目设计的奖励函数如下：

- **+10**: 吃到食物（正向激励）
- **-10**: 游戏结束/死亡（负向惩罚）
- **-0.01**: 每一步（小的负奖励，鼓励快速吃到食物）

这种设计平衡了即时奖励和长期目标：大的正奖励鼓励吃食物，大的负奖励惩罚死亡，小的负奖励避免智能体原地打转。

## 3. Q-Learning实现 (约400字)

### 3.1 算法原理

Q-Learning是一种基于值迭代的强化学习算法，核心是学习一个Q函数（Q-function），表示在状态s下执行动作a的期望累积奖励。

**Q-Learning更新公式**：

```
Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]
```

其中：
- Q(s,a): 当前状态s下执行动作a的Q值
- α (alpha): 学习率，控制更新步长
- r: 即时奖励
- γ (gamma): 折扣因子，权衡即时奖励与未来奖励
- s': 下一个状态
- max Q(s',a'): 下一状态的最大Q值

### 3.2 超参数选择

本项目使用的超参数配置：

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 学习率 (α) | 0.1 | 较小的学习率保证稳定学习 |
| 折扣因子 (γ) | 0.95 | 重视长期奖励 |
| 初始探索率 (ε) | 1.0 | 开始时完全探索 |
| 探索率衰减 | 0.995 | 每轮衰减0.5% |
| 最小探索率 | 0.01 | 保留1%的探索 |
| 训练轮数 | 500 | 平衡训练时间和效果 |

**超参数调优过程**：
- 学习率：尝试了0.05、0.1、0.2，最终选择0.1（收敛速度和稳定性的平衡）
- 折扣因子：尝试了0.9、0.95、0.99，选择0.95（较好地平衡即时和长期奖励）
- Epsilon衰减：尝试了0.99、0.995、0.998，选择0.995（约在200-300轮达到较小值）

### 3.3 状态编码方式

使用Python的`tuple`将状态数组转换为可哈希的键，存储在字典中：

```python
state_key = tuple(state)  # 例如: (0,0,0,1,0,0,0,1,0,0,1)
self.q_table[state_key] = [Q1, Q2, Q3]  # 三个动作的Q值
```

这种方式的优点是：(1) 查询速度快O(1)；(2) 只存储访问过的状态，节省内存；(3) 实现简单。

### 3.4 训练策略

**Epsilon-Greedy探索策略**：

```python
if random() < epsilon:
    action = random_action()  # 探索
else:
    action = argmax(Q[state])  # 利用
```

训练前期（epsilon高）以探索为主，积累经验；训练后期（epsilon低）以利用为主，优化策略。

**训练流程**：
1. 重置环境，获取初始状态
2. 循环执行：选择动作 → 执行动作 → 获取奖励 → 更新Q值
3. 游戏结束后衰减epsilon
4. 记录得分、epsilon等数据
5. 重复直到完成所有训练轮数

## 5. 挑战与解决方案 (约200字)

### 5.1 主要挑战

**挑战1：奖励稀疏问题**

问题描述：在初期训练中，智能体很难吃到食物，导致正奖励稀疏，学习缓慢。

解决方案：
- 添加小的负奖励（-0.01每步），鼓励快速行动
- 考虑过添加"距离奖励"（靠近食物+0.1），但发现会导致局部最优，最终放弃
- 适当延长训练轮数（500轮）确保充分学习

**挑战2：状态空间设计**

问题描述：如何用有限维度表示游戏状态是关键难题。

解决方案：
- 使用相对位置而非绝对坐标（减少状态数）
- 只关注关键信息：危险、方向、食物位置
- 通过实验验证11维状态足以支持有效学习

**挑战3：探索与利用的平衡**

问题描述：过早利用导致陷入局部最优，过度探索影响收敛速度。

解决方案：
- 采用指数衰减的epsilon（0.995每轮）
- 设置最小epsilon（0.01）保留少量探索
- 通过训练曲线监控调整衰减率

**挑战4：训练不稳定**

问题描述：初期训练时得分波动大。

解决方案：
- 使用较小的学习率（0.1）保证稳定性
- 计算100轮移动平均评估真实表现
- 保存最佳模型而非最后模型

### 5.2 技术难点

**Q表存储**：使用Python字典+tuple键的方式，既节省内存又保证查询效率。

**无限循环处理**：设置最大步数限制（100×蛇长），防止智能体原地打转。
